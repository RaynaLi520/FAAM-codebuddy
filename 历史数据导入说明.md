# FAAM 历史数据导入说明

## 概述

本工具用于导入历史爬取的Excel数据文件到FAAM数据库中。

## 使用方法

### 方法1: 使用批处理脚本(推荐)

双击运行 `import_history.bat`

### 方法2: 手动运行

```bash
cd D:\CodeBuddy\FAAM
python import_history.py
```

## 文件命名建议

为了正确识别商品上架日期，建议使用以下命名格式：

```
FAAM_Data_YYYYMMDD_HHMMSS.xlsx
```

例如：
- `FAAM_Data_20260415_183000.xlsx` - 2026年4月15日18:30的数据
- `FAAM_Data_20260410.xlsx` - 2026年4月10日的数据

系统会自动从文件名中提取日期作为新品检测日期。

## 导入流程

1. **检查数据库**: 如果数据库不存在，自动创建
2. **查找文件**: 在 `data/` 目录下查找所有 `.xlsx` 文件
3. **读取数据**: 逐个读取Excel文件
4. **识别新品**: 通过TCIN对比现有数据，识别新品
5. **导入数据**: 将数据插入/更新到 products 表
6. **记录新品**: 将新品记录到 daily_new_arrivals 表
7. **归档文件**: 将已处理的文件移动到 `data/archive/` 目录
8. **生成报告**: 输出导入统计信息

## 新品识别逻辑

系统通过以下方式识别新品：

1. 检查商品的TCIN是否已存在于数据库
2. 如果TCIN不存在，则认为是新品
3. 新品会同时记录到 `daily_new_arrivals` 表
4. 新品的 `商品标签` 字段会自动设置为 "Yes"

## 常见问题

### Q: 导入时出现错误怎么办？
A: 检查以下几点：
1. Excel文件是否被其他程序占用
2. 文件格式是否为 .xlsx
3. 必要的列是否都存在
4. 数据类型是否正确

### Q: 导入后数据没有更新？
A: 可能原因：
1. 数据文件已在 archive 目录（已被处理过）
2. TCIN已存在（会被更新而非新增）
3. 数据库文件没有写入权限

### Q: 如何重新导入某个文件？
A: 将文件从 `data/archive/` 移回 `data/` 目录，然后重新运行导入脚本。

### Q: 导入的新品日期不对？
A: 系统会从文件名提取日期。如果文件名不包含日期，会使用当前日期。请确保文件命名包含正确的日期。

## 数据备份

建议定期备份以下文件：
- `faam_products.db` - 数据库文件
- `data/archive/` - 所有历史数据文件

## 注意事项

1. 导入过程会修改数据库文件，请提前备份
2. 已处理的文件会自动移动到 archive 目录
3. 临时文件（以 `~$` 开头的文件）会被自动忽略
4. 导入过程中请勿关闭程序或关闭电脑
